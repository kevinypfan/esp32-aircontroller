### 電容替代方案

如果只買到 47µF 16V，可以**並聯 3~4 顆**達到接近 470µF 的效果：
- 3 顆並聯 = 141µF
- 4 顆並聯 = 188µF

並聯接法：所有正極接 5V，所有負極接 GND。**不要只用 1 顆 47µF**——緩衝量不夠，ESP 容易 brown-out 重開。

## CN3 Pinout

⚠️ **每台冷氣 PCB 標號方向不一定一樣！必須用三用電表實測確認！**

### 本機（MAXE RA-50SH32）實測結果

| Pin（由上而下）| 訊號 | 確認方式 | 接到 |
|---|---|---|---|
| 1 | **GND** | 量到 0V | BSS138 GND + ESP GND + 電容 − |
| 2 | TX (推測) | 5V 靜止（pull-up） | BSS138 HV1 |
| 3 | RX (推測) | 5V 靜止（pull-up） | BSS138 HV2 |
| 4 | **+5V** | ESP 接上後成功啟動 + 上 WiFi | BSS138 HV + ESP 5V + 電容 + |

**注意**：這跟 Midea 慣例（pin 1 = +5V, pin 2 = GND, pin 3 = TX, pin 4 = RX）**完全反過來**。OEM 機種正常會這樣，不是你接錯。

### Pin 量測步驟

冷氣**通電狀態**下，三用電表設成 DC 電壓檔，黑棒接機殼地（任何露出的金屬螺絲、機殼鐵殼都是 GND 參考）。

紅棒輪流碰 4 根 pin，記下讀數：

| 讀數 | 是什麼 |
|---|---|
| **0V** | GND |
| **微微抖動的 ~5V**（不是穩定靜止） | **+5V** ← 冷氣 PSU 在供電，AC MCU 跑起來會週期性吃電造成 ripple |
| **紋風不動的 ~5V** | TX / RX（兩根都會這樣，因為訊號 pin 被 pull-up 到 5V，靜止時沒人用）|

⚠️ **常見誤判**：「最穩定 5V 的」反而**不是**真正的 +5V！訊號 pin 被 pull-up 看起來最乾淨，但後面只接 10kΩ 電阻，最多供 0.5mA 電流，**接到 ESP 上 LED 會亮但 WiFi 一啟動就 brown-out**。真正的 +5V 因為 MCU 在吃電所以會有 ripple。

### TX vs RX 怎麼分

兩根都是靜止 5V 很難用電表分。最簡單做法：

1. 隨便猜一個方向接到 BSS138（HV1/HV2 對應 yaml 的 GPIO20/21）
2. 燒 ESP，看 log
3. 如果一直 `Response timeout` → yaml 對調 `tx_pin` 跟 `rx_pin`，OTA 重燒（**不用拆線**）

TX/RX 接反**不會燒**，只是收不到資料。

```yaml
# 如果第一次接反，把下面兩個對調即可
uart:
  tx_pin: GPIO21   # ← 對調成 GPIO20
  rx_pin: GPIO20   # ← 對調成 GPIO21
```

## ESP32-C3 SuperMini 腳位

```
                ┌──────────────┐
              5V│ ●          ● │GND
             3V3│ ●          ● │5  (GPIO5)
        (NC) GND│ ●          ● │6  (GPIO6)
       (GPIO4)4 │ ●          ● │7  (GPIO7)
       (GPIO3)3 │ ●          ● │8  (GPIO8)
       (GPIO2)2 │ ●          ● │9  (GPIO9)
       (GPIO1)1 │ ●          ● │10 (GPIO10)
       (GPIO0)0 │ ●          ● │20 (GPIO20, UART0 RX)
                │            ● │21 (GPIO21, UART0 TX)
                └──[ USB-C ]──┘
```

本專案使用：
- **5V** → 接 CN3 pin 1（+5V）
- **GND** → 接 CN3 pin 2（GND）
- **3V3** → 接 BSS138 LV（level shifter 的參考電壓）
- **GPIO20** → 接 BSS138 LV1（UART RX，接收冷氣資料）
- **GPIO21** → 接 BSS138 LV2（UART TX，送指令給冷氣）

---

## BSS138 4-channel 模組腳位

實物正面長這樣（HV/GND 在中間，HV1~4 跟 LV1~4 分散在兩側）：

```
        HV 側 (5V 側, 接冷氣)
    +-----------------------------+
    | HV1  HV2  HV   GND  HV3  HV4|
    |  o    o   o    o    o    o  |
    |                             |
    |  [ BSS138 x 4 MOSFET ]      |
    |                             |
    |  o    o   o    o    o    o  |
    | LV1  LV2  LV   GND  LV3  LV4|
    +-----------------------------+
        LV 側 (3.3V 側, 接 ESP)
```

注意排列：**HV1/HV2 在左邊、HV/GND 在中間、HV3/HV4 在右邊**。LV 那排也是對應位置。

本專案只用通道 1 跟 2（最左兩個），HV3/HV4 跟 LV3/LV4 完全不接。

## 接線圖

### 完整連線（分成 3 條獨立迴路，每張只看一件事）

#### A. 電源迴路（+5V / GND，並聯電容）

```
   CN3 Pin 4 +5V  ──┬──────────────────────┬──────  ESP 5V
                    │                      │
                    │                  +───┴───+
                    │                  │  +    │   (長腳 = 正極)
                BSS138 HV              │ 470µF │
                                       │  −    │   (白條 = 負極)
                                       +───┬───+
                                           │
   CN3 Pin 1 GND  ──┬──────────────────────┴──────  ESP GND
                    │
                BSS138 GND
```

#### B. 訊號迴路（UART，BSS138 做 5V ↔ 3.3V 雙向轉換）

```
   CN3 Pin 2 ────→  BSS138 HV1  ┃  BSS138 LV1  ────→  ESP GPIO20
   (冷氣 TX 5V)                  ┃                       (ESP RX 3.3V)
                              [電平轉換]
   CN3 Pin 3 ←────  BSS138 HV2  ┃  BSS138 LV2  ←────  ESP GPIO21
   (冷氣 RX 5V)                  ┃                       (ESP TX 3.3V)
```

#### C. BSS138 LV 側參考電壓

```
   ESP 3V3  ──────→  BSS138 LV  (告訴 BSS138 LV 側目標電壓是 3.3V)
```

> 三張圖加起來就是完整接線。實際接線照下面的「**接線表**」一條一條拉最不會錯。

### 接線表（最重要，照這個拉線就好）

⚠️ 下表 pin 編號照 **本機實測**（pin 1=GND, pin 4=+5V），不是 Midea 慣例。

| 線編號 | 從 | 到 | 訊號 | 備註 |
|---|---|---|---|---|
| ① | CN3 **pin 4** | BSS138 **HV** | +5V | 同時接 ESP **5V** 跟電容 **+** |
| ② | CN3 **pin 1** | BSS138 **GND**（任一）| GND | 同時接 ESP **GND** 跟電容 **−** |
| ③ | CN3 **pin 2** | BSS138 **HV1** | TX (5V) | 冷氣 → ESP 方向（TX/RX 待驗證）|
| ④ | CN3 **pin 3** | BSS138 **HV2** | RX (5V) | ESP → 冷氣 方向（TX/RX 待驗證）|
| ⑤ | ESP **3V3** | BSS138 **LV** | 3.3V ref | level shifter 知道要轉到 3.3V |
| ⑥ | BSS138 **LV1** | ESP **GPIO20** | TX (3.3V) | 對應 ③ 降壓後 |
| ⑦ | BSS138 **LV2** | ESP **GPIO21** | RX (3.3V) | 對應 ④ 升壓前 |
| ⑧ | ESP **5V** | 電容 **正極** | — | 直接焊在 ESP 5V pin 上 |
| ⑨ | ESP **GND** | 電容 **負極** | — | 直接焊在 ESP GND pin 上 |

**注意**：BSS138 上面 HV 排有 GND、下面 LV 排也有 GND，兩個內部相通，**只接一個就好**（圖上 ② 接哪邊都可以）。

### 電容怎麼接（放大）

電容是**橫跨**在 5V 跟 GND 之間：

```
                   +-------+
   ESP 5V pin ----+|  +    |     <-- 長腳，正極接 5V
                  |  470µF |
                  |        |
   ESP GND pin ---+|  -    |     <-- 短腳/有白條那側，負極接 GND
                   +-------+
```

**並聯多顆**（用 47µF 湊的話 3~4 顆）：

```
   ESP 5V  -----+-------+-------+--
                |       |       |
              +---+   +---+   +---+
              |+ -|   |+ -|   |+ -|       三顆都長腳朝上、白條朝下
              |47 |   |47 |   |47 |
              |µF |   |µF |   |µF |
              +---+   +---+   +---+
                |       |       |
   ESP GND -----+-------+-------+--
```

### 物理焊接位置

最直接：**電容兩腳跨焊到 ESP32-C3 SuperMini 背面的 5V 跟 GND 焊盤上**：

```
    ESP32-C3 SuperMini 背面
    +-----------------------+
    | 5V pad         GND pad|
    |  *              *     |   <-- 焊盤
    |  |              |     |
    |  +--+        +--+     |
    |     |        |        |
    |   +-+--+   +-+--+     |   電容立著焊，兩腳跨在 5V 跟 GND 焊盤
    |   | +  |   |  − |     |   長腳/無白條 → 5V
    |   |470 |   | µF |     |   短腳/有白條 → GND
    |   +----+   +----+     |
    +-----------------------+
```

不會焊 PCB 的話，**麵包板也行**：

- 把 5V 跟 GND 拉到麵包板兩條軌道
- 電容插在麵包板上跨那兩條軌道
- 但**整段 5V 線（從 CN3 到 ESP）總長 < 10cm**，越短越好——線越長電感越大，電容效果就被抵消

### 極性提醒（再次）

⚠️ 接反會爆，第一次通電前再三檢查：

| 電容那一側 | 接哪 |
|---|---|
| **長腳** / 沒有白條紋那一側 | +5V（**正極**）|
| **短腳** / 有白色直條紋那一側 | GND（**負極**）|

---

## 電容極性！⚠️

470µF 是**有極性**的電解電容，接反會爆：

- **長腳 / 不在白條那側 = 正極（+）** → 接 +5V
- **短腳 / 白條那側 = 負極（−）** → 接 GND

電容外殼上會有一條白色或銀色直條紋指向**負極**那一側。第一次通電前一定要再三檢查。

## 為什麼需要這些元件

| 元件 | 角色 | 不裝的後果 |
|---|---|---|
| BSS138 | 5V ↔ 3.3V 雙向電平轉換 | 5V 灌進 ESP32-C3 GPIO，慢性損壞或當場燒掉 |
| 470µF 電容 | 5V 電源緩衝 | 壓縮機啟動瞬間 5V 跌落，ESP brown-out 重開 |
| 熱縮管 | 包覆裸線焊點 | 短路可能燒 ESP 或冷氣主板 |
| JST XH 線 | 對接 CN3 公座 | 不對應規格的接頭插不進去 |

## 物理安裝建議

- **位置**：不要黏在金屬背板上，WiFi 訊號會被擋；盡量靠近塑膠面板或出風口
- **遠離結霜區**：避開蒸發器翅片（會結霜流水）
- **走線**：CN3 線不要跟主板上的 220V 強電線並排走，平行至少間隔 1cm，最好用屏蔽線
- **固定**：用雙面膠或束帶固定 ESP+BSS138，避免被冷氣震動晃到金屬處短路
- **三防漆**（選配）：跑穩後塗 PCB 三防漆（蝦皮搜「PCB 三防漆 噴罐」）防潮，特別是冷氣會凝結水
